# MC-LIVE-REFRESH-1: fix no-auto-refresh bug + add live heartbeat

## Root cause
- `grep -n setInterval 01_projects/mission-control/code/mission-control.html` → 0 hits
- The footer text says "auto-refresh: 30s" but there is NO setInterval. The page only refreshes on manual click.
- `data_agents()` derives `last_activity` from log-file mtimes in `00_company_os/04_agents/logs/`. Thor/Forge/Argus don't write log files for chat work — only for subagent work. So even if the page DID refresh, chat-active agents would still show "8h ago" or whatever the last subagent-log mtime is.

  - New API: `POST /api/heartbeat` with body `{"agent": "thor|forge|argus"}` (default `thor` if missing). Writes `00_company_os/04_agents/logs/.heartbeat-<oid>` with `{"agent": "...", "ts": "..."}`. No auth required (LAN write, not sensitive).

  - 5+ new tests, all must pass.
