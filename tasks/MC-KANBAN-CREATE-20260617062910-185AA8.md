### 1. Smart plant watering system 🌱
- **Difficulty:** Beginner
- **Components:** ESP32 DevKit V1, capacitive soil-moisture sensor (v2), 5 V peristaltic pump or 12 V solenoid valve, IRLZ44N MOSFET, 5 V / 12 V PSU, optional 0.96" OLED.
- **Description:** Reads soil moisture every 10 minutes. If below a threshold, drives the pump for a fixed number of seconds. Exposes a simple web UI on the local network so you can override or water manually.
- **Why it's a good home project:** No safety risk, instant visible payoff (a plant that doesn't die when you travel), parts are sub-$30, and it teaches the full input → decision → output loop. Great first ESP32 build for someone who has only used Arduino.
- **Stretches:** Add BME280 for ambient temp/humidity compensation; add Home Assistant via MQTT; add a water-level float switch so the pump doesn't run dry.

### 2. Indoor weather station with BME280 + TFT 🌦️
- **Difficulty:** Beginner
- **Components:** ESP32, Adafruit BME280 (already in the DIY Hub V1 catalog), 3.5" ILI9488 + XPT2046 TFT (also in catalog), LiPo + TP4056 charger (also in catalog), 3D-printed or laser-cut enclosure.
- **Description:** Reads temperature, humidity, and barometric pressure from the BME280 and renders a clean dashboard on the TFT (current readings, 24 h min/max sparkline, weather icon derived from pressure trend). Optionally publishes the same data over MQTT.
- **Why it's a good home project:** Reuses three components NOFI has already validated, looks great on a shelf or wall, and teaches I²C + SPI on the same board. ILI9488 + XPT2046 touch is a known-supported combo in PlatformIO, so the firmware side is well-trodden.
- **Stretches:** Add a small e-ink panel (lower power, longer battery life); add an outdoor sensor over LoRa; render a 3-day forecast pulled from Open-Meteo (free, no key).

### 3. LED matrix notification board 📢
- **Difficulty:** Intermediate
- **Components:** ESP32, 64×32 HUB75 RGB LED matrix, 5 V / 4 A PSU, 3D-printed frame, optional 3.5 mm jack-to-DIN cable. Code: ESP32-HUB75-MatrixPanel-DMA library.
- **Description:** A wall-mounted panel that scrolls calendar events, weather, package-tracking numbers, build status from GitHub Actions, or stock prices. Configured over Wi-Fi via a captive portal, no app needed.
- **Why it's a good home project:** It's the kind of thing people see in real offices and assume is expensive. Under $50 in parts. It teaches DMA-driven display driving, Wi-Fi, and JSON parsing. High "wow-per-dollar" ratio.
- **Stretches:** Drive two panels daisy-chained; add a small PIR sensor to wake the display on motion; add a microphone for sound-reactive animations on the weekend.

### 4. MQTT home-automation sensor hub 🏠
- **Difficulty:** Intermediate
- **Components:** Raspberry Pi 4 (or any old x86 box) running Mosquitto + Home Assistant; 3–5 ESP32 nodes, each with a DHT22 / BME280 / door-switch / PIR; one Wemos D1 Mini acting as an IR blaster for the AC.
- **Description:** The Pi runs the broker and a Home Assistant instance. Each ESP32 publishes JSON over MQTT (`home/kitchen/temp`) every 30 s. Automations turn on the AC when the bedroom hits 27 °C, flash the office light when CI fails, etc.
- **Why it's a good home project:** Modular — you can ship one node a weekend and grow the system. No cloud, no subscription, no data leaving the house. Becomes the substrate for projects 1, 2, 5, 6, 7, 9, 10 below — once you have MQTT, every new sensor is a 2-hour build.
- **Stretches:** Add Node-RED for visual automations; add Tasmota-flashed smart plugs; build a small e-ink "now playing" panel in the kitchen.

### 5. Air-quality + CO₂ monitor 🌬️
- **Difficulty:** Intermediate
- **Components:** ESP32, Sensirion SCD30 (NDIR CO₂ + temp + humidity), Plantower PMS5003 (PM1.0/2.5/10), optional BME280 for pressure, 1.8" or 3.5" TFT.
- **Description:** A bedside/desktop box that displays CO₂ in ppm, PM2.5 in µg/m³, temperature and humidity, with traffic-light color coding (green / amber / red). Logs to SD card and publishes over MQTT. CO₂ above 1000 ppm is the threshold most HVAC guides use to open a window.
- **Why it's a good home project:** Indoor air quality is the kind of thing people only notice when it's bad. Once you see the CO₂ climb to 1800 ppm in a closed bedroom, you never go back. Hardware is well-supported by Adafruit and Sensirion's own libraries, and the data is genuinely useful.
- **Stretches:** Add a small buzzer that chirps over 1500 ppm; add a graph of the last 24 h to spot bedroom-ventilation patterns.

### 6. Sous-vide / PID temperature controller 🍳
- **Difficulty:** Intermediate
- **Components:** ESP32, K-type thermocouple + MAX6675 amplifier, 5 V relay or 25 A SSR, mains-rated outlet, 1.3" OLED + rotary encoder. Code: Arduino PID library + autotune.
- **Description:** A drop-in PID controller for a slow cooker / rice cooker / immersion circulator. Set a target temperature (e.g. 56 °C for 4 h for steak), the box cycles the heater on/off via the SSR, logs the actual vs. setpoint curve to a phone-accessible web UI, and beeps when done.
- **Why it's a good home project:** Genuinely useful in the kitchen — the cook quality of a PID-driven sous-vide is several notches above a $30 ANOVA knockoff. The PID math is the same as in industrial process control, so it's a great teacher. Safety note: the box only switches the live wire through the SSR; the esp32 is on the low-voltage side and is fully isolated.
- **Stretches:** Add Wi-Fi + MQTT; add a profile library (egg, salmon, brisket); add a thermal cutoff fuse for belt-and-braces safety.

### 7. Smart doorbell with camera (no cloud) 🔔
- **Difficulty:** Advanced
- **Components:** ESP32-S3 with PSRAM (e.g. ESP32-S3 DevKitC-1 or Freenove), OV2640 camera module, button, small speaker + amp, microSD, 3D-printed enclosure, optional WS2812 ring for status. Code: ESPHome or Arduino + esp32-camera + ESP-Video.
- **Description:** When the button is pressed, the ESP32-S3 captures a still or short MJPEG, saves it to the SD card, and pushes a notification (with the image) to your phone via MQTT + a small companion app (or Telegram bot, free). The "no cloud" part is the point — no monthly fee, no Eufy / Ring privacy concerns, footage stays on the LAN.
- **Why it's a good home project:** Cameras are intimidating because of the lens, the buffer, the network, and the power. Once you've shipped one you can build any vision project. The ESP32-S3 has USB-OTG and enough PSRAM for 1280×720 capture, and the ESPHome firmware is mature enough that you're really gluing blocks together.
- **Stretches:** Add face recognition with on-device TFLite; add two-way audio; replace the SD with a small NAS share over SMB.

### 8. Plant-growth monitor with SD logging 🪴
- **Difficulty:** Beginner
- **Components:** ESP32, 3–4 capacitive soil-moisture sensors (one per pot), BME280, DS3231 RTC, microSD breakout, 18650 + TP4056.
- **Description:** Placed next to a row of plants, it logs moisture, ambient temp/humidity, and a wall-clock timestamp to a CSV on the SD card every 5 minutes. Drop the SD into a laptop and you get a months-long view of how each plant's needs change with the season.
- **Why it's a good home project:** Essentially a data-acquisition box — the kind of thing used in real agronomy research, shrunk to $30 and a breadboard. It teaches I²C, SPI, RTCs, and CSV writing. The data is genuinely interesting: you'll see watering patterns you never noticed.
- **Stretches:** Add a small e-ink summary panel; add a TP-link smart-plug control so the system can also drive a pump (becomes project 1); graph the data in Grafana on the home Pi.

### 9. Pet feeder with servo + RTC schedule 🐶
- **Difficulty:** Intermediate
- **Components:** ESP32, MG996R high-torque servo (or a small 12 V auger motor), DS3231 RTC, tactile buttons + OLED for manual feed, optional load cell + HX711 to detect "did the food actually come out?".
- **Description:** Schedule one or two feeds a day from a phone UI. At the scheduled time, the servo turns a hopper 90°, dropping a measured portion. The RTC keeps time even if Wi-Fi dies; manual buttons on the unit let you feed without the app.
- **Why it's a good home project:** The servo + RTC + load-cell combo is classic embedded work, and a pet feeder is the kind of thing that, once it works, gets used every day for years. The load cell is optional but worth it: it catches the "hopper empty" failure mode that otherwise leaves your cat hungry.
- **Stretches:** Add a camera (becomes project 7 lite); add a webhook that pulls feeding history into Notion; add a second hopper for meds.

### 10. Pool / aquarium temperature + pH monitor 🐟
- **Difficulty:** Intermediate
- **Components:** ESP32, waterproof DS18B20 (1-Wire), Atlas Scientific pH probe + EZO carrier board, 5 V pump for the pH water loop, optional TDS probe, OLED or TFT.
- **Description:** Two sensors, one board, one place. The DS18B20 goes in the water and reports temperature every minute. The pH probe lives in a small flow cell with a pump that pulls aquarium water past it. Both readings are logged to SD and pushed over MQTT. Threshold alerts (e.g. pH > 8.2 in a freshwater tank) light up an LED and send a notification.
- **Why it's a good home project:** pH probes are intimidating because they look like chemistry lab gear, but the EZO carrier board handles calibration and the I²C protocol is well-documented. Once you have it logging, you can correlate pH drift with CO₂ injection, water changes, plant growth, etc. — the same kind of data work as project 8, but wet.
- **Stretches:** Add a small peristaltic pump that doses pH-down on a PID loop; add a TDS / salinity probe; add a heater control via SSR (crosses over with project 6).
